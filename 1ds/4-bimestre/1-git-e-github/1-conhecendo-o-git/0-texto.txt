O QUE É GIT?
Git é um software livre e de código aberto para controle de versão.
Ele ajuda os desenvolvedores a colaborar entre si, guardar um histórico de alterações e reverter para versões anteriores se necessário.

MAS O QUE É CONTROLE DE VERSÃO?
O monitoramento de todas as alterações em cada arquivo de um projeto.
Conjuntos de alterações podem ser salvas em versões: 1.0.1, 1.0.2, 1.2.0, etc.

POR QUE O VERSIONAMENTO É IMPORTANTE?
Ajuda a reverter o projeto para uma versão anterior em caso de problemas.
Ajuda a controlar e organizar todas as modificações feitas pelos desenvolvedores (especialmente em equipes grandes).

CONCEITOS FUNDAMENTAIS DO GIT
Repositório: como nos referimos ao projeto como um todo, o diretório raiz a partir de onde serão monitoradas todas as alterações.

Commit: é o ato de salvar uma ou mais alterações no código e também pode se referir ao conjunto dessas alterações.
    Do inglês, commit significa "se comprometer", o que é apropriado para o ato de salvar as alterações no repositório de forma permanente.

Working Tree: é o espaço de trabalho onde você faz modificações nos arquivos do seu projeto. Quando você edita, cria ou deleta arquivos, essas mudanças acontecem na working tree.

Staging: é a área de preparação onde as alterações são separadas para serem salvas em um commit, ou seja, elas permitem escolher exatamente o que será salvo.

Branches: são ramificações que podemos criar a partir de um ponto no histórico do projeto para trabalhar paralelamente, assim é possível trabalhar sem afetar a ramificação principal.
    Todo repositório precisa de pelo menos uma branch, para servir como principa (geralmente chamada de 'main' ou 'master').
